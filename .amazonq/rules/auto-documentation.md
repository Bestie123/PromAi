# Автоматическая документация и промпты

## Правило: При добавлении нового функционала ВСЕГДА создавай:

### 1. Промпт для работы с функционалом
**Файл:** `.amazonq/prompts/PROMPT_{FunctionalityName}.md`

### 2. Обновление правил проекта
**Файл:** `.amazonq/rules/project-standards.md`

### 3. Документация использования
**Файл:** `docs/{FunctionalityName}_GUIDE.md`

### 4. Обновление README.md
Добавить секцию о новом функционале

### 5. Обновление FULL_PROMPTS_1-12.md
Добавить новый промпт если нужен

## Автоматические обновления при добавлении функционала

### Обязательно обновить:
1. ✅ `project_registry.json` - добавить модуль/функции
2. ✅ `dependencies_map.json` - добавить зависимости
3. ✅ `function_registry.json` - добавить функции с вызовами
4. ✅ `todo.json` - создать задачи
5. ✅ `changelog.md` - записать изменения
6. ✅ `README.md` - обновить секцию возможностей
7. ✅ `.amazonq/rules/project-standards.md` - добавить правила
8. ✅ `.amazonq/prompts/PROMPT_{Name}.md` - создать промпт
9. ✅ `docs/{Name}_GUIDE.md` - создать документацию

### Опционально обновить:
- `FULL_PROMPTS_1-12.md` - если нужен новый промпт
- `ARCHITECTURE_DIAGRAM.md` - если изменилась архитектура
